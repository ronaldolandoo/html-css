------------------------------------O QUE É GIT E GITHUB---------------------------
	REPOSITÓRIO: É UM LUGAR QUE VOCÊ GUARDA COISAS (CÓDIGOS, NO SEU CASO).
	
	GIT: CRIAR VERSÕES. /*REPOSITÓRIO LOCAL*/
	GITHUB: BACKUP DAS VERSÕES DO GIT. /*REPOSITÓRIO REMOTO*/

	NOTE: COM O GIT (REPOSITÓRIO LOCAL) E GITHUB (REPOSITÓRIO REMOTO), VOCÊ PODE TRABALHAR EM EQUIPE, CADA DESENVOLVEDOR DEVE TER O SEU GIT (REPOSITÓRIO LOCAL) E PODERÁ LANÇAR VERSÕES (FAZER PUSH) NO GITHUB (REPOSITÓRIO REMOTO).

	NOTE: QUANDO TIVERES MAIS EXPERIÊNCIA ESTUDE, GIT E GITHUB COM MAIS DETALHES.

	# SCM = SOURCE CODE MANAGER

	# COM O GITHUB DESKTOP, VOCÊ NÃO PRECISAR SABER MUITOS COMANDOS DO GITHUB.

	# GIT, É UM GESTOR DE CÓDIGO.

	# NAS PASTAS O ".git" significa nada mais do que uma cópia do seu código.

	# LÓGICA - GIT DESKTOP VS GITHUB
	- GIT DESKTOP: cria backups localmente
	- GITHUB (PUSH): cria backups remoto (no servidor)
